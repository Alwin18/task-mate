# Task Mate
Develop by Go with Domain-Driven Design (DDD) architecture that uses Swagger API. The application includes employee attendance, job management, and employee profile functions.

# Structure Folder
```
    /project-root
    ├── /api
    │   ├── /swagger
    │   │   ├── swagger.yaml
    │   │   └── docs.go
    │   └── /handlers
    │       ├── employee_handler.go
    │       ├── role_handler.go
    │       └── attendance_handler.go
    ├── /cmd
    │   └── /server
    │       └── main.go
    ├── /config
    │   └── config.go
    ├── /domain
    │   ├── /employee
    │   │   ├── employee.go
    │   │   └── repository.go
    │   ├── /role
    │   │   ├── role.go
    │   │   └── repository.go
    │   └── /attendance
    │       ├── attendance.go
    │       └── repository.go
    ├── /infrastructure
    │   ├── /persistence
    │   │   ├── employee_repository.go
    │   │   ├── role_repository.go
    │   │   └── attendance_repository.go
    │   ├── /http
    │   │   └── router.go
    │   └── /db
    │       └── db.go
    ├── /interfaces
    │   ├── /employee
    │   │   ├── employee_service.go
    │   │   └── employee_controller.go
    │   ├── /role
    │   │   ├── role_service.go
    │   │   └── role_controller.go
    │   └── /attendance
    │       ├── attendance_service.go
    │       └── attendance_controller.go
    ├── /usecase
    │   ├── /employee
    │   │   ├── employee_usecase.go
    │   │   └── employee_usecase_test.go
    │   ├── /role
    │   │   ├── role_usecase.go
    │   │   └── role_usecase_test.go
    │   └── /attendance
    │       ├── attendance_usecase.go
    │       └── attendance_usecase_test.go
    └── go.mod
```
* /api: Stores all files related to the API, including Swagger definitions and handlers.
  - /swagger: Stores Swagger definition files (swagger.yaml) and files generated by Swagger (docs.go).
  - /handlers: Stores HTTP handlers for each domain.
* /cmd/server: Stores the main entry point for the application, such as the main.go file.
* /config: Stores the application configuration, such as the config.go file.
* /domain: Stores the core logic of the application, in accordance with the Domain-Driven Design concept.
* /infrastructure: Stores the infrastructure implementation, such as databases and HTTP routers.
  - /persistence: Repository implementation for the model domain.
  - /http: Manages HTTP routing.
  - /db: Database connection and settings.
* /interfaces: Stores the service and controller implementations for the model domain.
* go.mod: Go module file that manages dependencies.
